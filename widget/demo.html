Local widget demo

Customers normally install the widget with a bare snippet — no key in the page — after registering their website domain in the dashboard (Install & API):

<script src="https://api.yourdomain.com/widget/chat.js" async></script>

Don't open this file directly from disk — file:// pages are isolated origins (browsers may even log "Unsafe attempt to load URL file:..." when previewed in an IDE frame), and they send no usable Origin, so domain matching can't work. Instead either:

1. Open the served preview: http://localhost:8080/widget/demo (keyless once localhost is registered as a domain in the dashboard), or

2. Serve this file over HTTP — python3 -m http.server 9000 in this folder, register localhost in the dashboard, open http://localhost:9000/demo.html, or

3. As a last resort for direct file opening, paste your widget key into the data-widget-key attribute below.
